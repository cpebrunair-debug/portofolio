1. Konsep Utama
Workbook PLN mendefinisikan HI untuk intensitas bahaya per lokasi unit dan DI untuk severitas kerusakan per skenario.
🧭 Hazard Index (HI)

HI berada pada rentang 0 - 1. Nilai ini dibaca langsung dari sheet 02_DATA_PEMBANGKIT untuk lima bencana: banjir, cuaca ekstrem, kebakaran hutan, kekeringan, dan tanah longsor.

Nilai HIHazard LevelTingkat RisikoInterpretasi untuk Pembangkit PLN
0No HazardTidak Ada RisikoTidak teridentifikasi berada di zona bahaya bencana tersebut.
0 - 0,20Very LowSangat RendahPaparan sangat kecil; dampak minimal pada operasi pembangkit.
0,20 - 0,40LowRendahRisiko terbatas; gangguan kecil dan pemulihan relatif cepat.
0,40 - 0,60ModerateSedangRisiko nyata; potensi derating atau shutdown parsial beberapa hari.
0,60 - 0,80HighTinggiKerusakan signifikan; kemungkinan penggantian komponen utama.
0,80 - 1,00Very HighSangat TinggiRisiko sangat besar; potensi total loss pada unit.
🛠️ Damage Index (DI)

Sheet parameter PLN mendefinisikan DI dengan menggabungkan kerusakan fisik, downtime index, dan faktor bencana. Nilai DI dibatasi maksimum 1.

Formula Damage Index
DI = MIN[(wD x Kerusakan + wT x DowntimeIndex) x FaktorBencana, 1]
wD = 0,5 - bobot komponen kerusakan fisik.
wT = 0,5 - bobot komponen downtime / kehilangan pendapatan.
DowntimeIndex = MIN(Downtime / 365, 1).
Formula Kerugian = Revenue Tahunan x HI x DI.

3. Cara Baca Dashboard
Versi HTML ini mempertahankan dua mode kerja agar konsisten dengan workbook PLN.
🔵 Model 1 - Revenue Terekspos

Untuk setiap bencana aktif, unit dengan HI > 0 diperlakukan sebagai unit terekspos. KPI dan ranking pada mode ini menampilkan revenue tahunan terekspos.

Formula Tampilan
Revenue Terekspos = Revenue Tahunan jika HI > 0, selain itu 0
🟣 Model 2 - Estimasi Kerugian

Mode ini menggunakan kombinasi HI dan DI dari tabel skenario untuk menghitung potensi kerugian finansial per unit pembangkit.

Formula Tampilan
Potensi Kerugian = Revenue Tahunan x HI x DI
Koordinat unit pada dashboard ini telah diperbarui dari file Koordinat Pembangkit.xlsx dan informasi kabupaten/kota juga diselaraskan dari file tersebut. 273 dari 291 unit berhasil dicocokkan langsung atau melalui alias nama lokasi; 18 unit yang belum memiliki padanan meyakinkan di file Excel tetap memakai data koordinat sebelumnya agar tidak salah lokasi.

04_MODEL2_KERUGIAN

Referensi ringkasan potensi kerugian, revenue terekspos, dan loss ratio per skenario.
